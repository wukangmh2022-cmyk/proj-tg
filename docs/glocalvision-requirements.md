# glocalVision Product Requirements (Agentic Telegram)

Last Updated: 2026-02-26
Status: Draft v0.1

## 1. 产品目标

打造一个以 Telegram 为核心信息源的 Agentic 应用，覆盖：

- 多群/多频道实时监控
- 市场情绪与主题热点识别
- 异常事件/重要新闻检测与提醒
- 基于 LLM 的结构化分析、翻译、触发策略
- 日报/周报自动产出与机会发现建议

定位：比原版 Telegram 更偏“投研与交易情报工作台”。

## 2. 核心用户提问范式（Agentic Query Patterns）

系统需要支持将自然语言问题编排为可执行流程：

1. 监控型
- 例：监控 XXX 土狗交流群，如果出现“大金狗”信号就提醒我。

2. 变化检测型
- 例：监控 XX 二级交流群，检测今日市场情绪变化（上涨/下跌/恐慌/FOMO）。

3. 定时总结型
- 例：每天 09:00 输出 XX、LL、MM 三个群的情绪日报和热门板块提及。

4. 机会发现型
- 例：基于过去 24h/7d 的讨论变化，找可能的获利机会并给证据。

5. 事件驱动型
- 例：出现“川普/马斯克喊单”或类似重大影响人物发言时，进行跨群影响评估。

## 3. 必备能力抽象

## 3.1 数据接入层

- Telegram 数据拉取（频道、群、消息、媒体、转发关系）
- 多来源扩展（可选）：openindex/openviking 检索适配器
- 历史数据导入与增量同步
- 去重与标准化（消息 ID、时间、语言、符号、实体）

## 3.2 检索与过滤层（轻检索优先）

- `rg/grep` + 关键词规则检索
- 布尔表达式过滤（AND/OR/NOT）
- 时间窗检索（1h/24h/7d/30d/2y）
- 角色过滤（KOL、管理员、普通用户）

## 3.3 语义理解层（LLM）

- 消息意图分类（信号、新闻、情绪、噪音）
- 情绪标签（看多/看空/中性/恐慌/FOMO）
- 热点主题抽取（链、板块、标的、叙事）
- 事件归因（哪个消息触发了情绪或价格预期变化）

## 3.4 策略与触发层

- 规则触发：关键词、提及次数突增、情绪阈值
- LLM 触发：基于语义理解命中条件（例如“疑似大金狗”）
- 异常触发：短时消息量激增、情绪反转、跨群同步异动
- 触发后动作：通知、置顶、写入日报、创建待办

## 3.5 分析与报表层

- 情绪日报（群级、主题级、标的级）
- 热门板块/标的排行榜（提及次数、增速）
- KOL 历史信号统计（胜率、累计收益、回撤、样本量）
- 机会清单（含证据与风险提示）

## 3.6 通知与交互层

- 自定义频道/群提醒配置
- 多级提醒（普通/重要/紧急）
- 触达渠道（App 内、系统通知、Webhook 可扩展）
- Agent 对话与可追溯证据展示

## 3.7 客户端增强能力（重要功能）

- 截屏限制开关（关闭截屏限制）
  - 需求：允许用户配置是否解除截图限制
  - 风险：涉及平台合规与隐私告知，需明确免责声明与权限提示
- LLM 翻译配置
  - 需求：可选模型、术语词表、目标语言、保留币种符号
- 自定义提醒配置
  - 规则提醒 + LLM 理解触发提醒

## 4. 详细功能需求清单

## 4.1 监控任务（Watchers）

- 创建/编辑/暂停/删除监控任务
- 输入：目标群列表、关键词/语义条件、时间窗、触发阈值
- 输出：命中事件流 + 摘要
- 任务模板：
  - 大金狗监控
  - 情绪反转监控
  - 重大人物喊单监控

## 4.2 情绪分析

- 指标：
  - 正负情绪占比
  - 情绪变化率（今日 vs 昨日 / 近7日均值）
  - 情绪分歧度（群内意见分散程度）
- 维度：按群、按主题、按标的、按KOL

## 4.3 热点与板块识别

- 提及频次 Top-N
- 新增热点（历史低频 -> 当日高频）
- 板块轮动检测（AI、MEME、L2、DeFi 等）

## 4.4 新闻/异常检测

- 重要人物实体识别（Trump, Musk 等可扩展列表）
- 异常消息流检测：
  - 单群爆发
  - 跨群共振
  - 突发高转发链路
- 输出：事件卡片（时间、来源、影响范围、可信度）

## 4.5 交易信号统计

- 入场/出场信号抽取
- 交易配对与绩效计算
- 缺失数据补全（可选：接交易所历史价格 API）
- 输出标准表格（可导出 Markdown/CSV）

## 4.6 日报自动化

- 调度：每日固定时间
- 输入群：XX/LL/MM 等可多选
- 内容最小集合：
  - 情绪摘要
  - 热门板块
  - 重要事件
  - 潜在机会 + 风险
- 输出：日报文本 + 数据表

## 4.7 多语言与翻译

- 自动检测消息语言
- 翻译策略：逐条翻译 / 摘要翻译
- 术语保护（Ticker、链名、人名不误译）

## 5. 非功能需求

- 可追溯：每个结论必须可回链到证据消息
- 低延迟：重要监控场景下分钟级响应
- 稳定性：任务失败可重试、断点续跑
- 可解释：LLM 结论需显示触发规则与置信度
- 配置化：绝大多数能力可通过配置调整

## 6. 风险与边界

- 数据真实性：群消息可能有噪音、反串、喊单操纵
- 绩效偏差：信号发布时间与实际成交价存在偏差
- 合规风险：投资建议、隐私、截图限制相关平台政策
- 模型幻觉：必须限制“无证据推断”

## 7. 分阶段开发路线图

## Phase 0（已具备基础）

- 轻检索信号统计（grep/规则）
- 基础收益与回撤统计
- Prompt 输出用于二次分析

## Phase 1（优先实现）

- 真实 Telegram 增量同步
- Watcher 任务系统（规则触发）
- 日报调度与模板输出
- 自定义群提醒

## Phase 2

- LLM 语义触发器
- 情绪与热点多维分析
- 重大事件检测（人物/跨群共振）
- LLM 翻译配置中心

## Phase 3

- openindex/openviking 检索适配
- 交易所价格校验与缺失补价
- 机会发现评分系统
- 客户端增强（截图限制开关等）

## 8. 技术架构建议（当前方向）

- Core: Rust（任务编排、规则引擎、统计引擎）
- Data: TDLib 接入 Telegram 数据流
- Retrieval: 本地轻检索优先，外部检索适配器为可选插件
- LLM: 可配置多模型（分类、摘要、翻译、触发判断）
- Storage:
  - 原始消息存储
  - 结构化特征存储（实体、情绪、主题、信号）
  - 报表存储

## 9. 配置中心（必须可视化）

- LLM Provider/API Key/模型路由
- 翻译规则（目标语言、术语保留）
- 提醒规则（触发条件、冷却时间、通知方式）
- 监控群列表与优先级
- 截图限制开关（含风险确认）

## 10. MVP 验收标准

- 给定 3 个目标群，能稳定生成日报
- 可检测当天情绪变化并给出量化指标
- 可对“疑似大金狗”给出触发提醒和证据链
- 可生成至少一份 KOL 信号统计表（含收益与回撤）
- 所有结论均可追溯到原始消息

## 11. 待你确认的关键产品决策

- 目标平台优先级：桌面端优先 / 移动端优先
- 通知优先级：App 内通知 / 系统推送 / 第三方Webhook
- 机会发现风格：激进（高召回）/ 稳健（高精度）
- 默认语言：中文 / 英文 / 双语
